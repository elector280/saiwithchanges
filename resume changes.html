1. Context and Original Problem

The public pages of the SAI Mini Campaigns were not correctly displaying the visual or functional elements configured from the administration panel. The problem was that the frontend Blade view was accessing the properties of the $cam object incorrectly, without considering that the model uses multi-language translation (Spatie Translatable) and that the image fields and donation widget need special processing before being displayed.

Reported symptom: The mini campaign "SAI Provides Food and Medical Care" showed the donation widget correctly (because it was pre-configured from the previous version), but the "Children Medical Clinic Venezuela" campaign showed the gray warning box: "No donation widget provided", and the cover image did not load.

3. Applied Corrections
🖼️ Fix #1 — Image Paths FIX

The cover image field (cover_image) and the OG image field (og_image) are saved in the database only as the filename (e.g., 1745259032_cover.jpg). The previous code tried to directly use $cam->cover_image_url as the full URL — a property that does not exist in the real database model and only exists during admin previews (where the controller manually creates that attribute).

Before (incorrect):
style="{{ !empty($cam->cover_image_url) ? 'background-image: url(...' : '' }}"

After (correct):
Resolved at the beginning of the file by building the full URL:
$cover_image_url = !empty($cam->cover_image) ? asset('storage/cover_image/' . $cam->cover_image) : null;
With backward compatibility for the admin preview using isset().

💳 Fix #2 — Donation Widget (Donation Box) FIX

The donation_box field is defined as a translatable (multi-language) field in the model with Spatie Translatable. This means it is saved in the database as a JSON object {"en": "<iframe>...</iframe>"}, and when accessed with $cam->donation_box it can return that full JSON as a string or an array depending on the context. The original template used it directly without unwrapping it, causing it to be interpreted as empty or showing the literal JSON.

@php // New logic — unwraps correctly regardless of language $donation_box = method_exists($cam, 'getLocalValue') ? $cam->getLocalValue('donation_box') : ($cam->donation_box ?? ''); // If it is still an array, extract the value of the active locale if (is_array($donation_box)) { $locale = app()->getLocale(); $donation_box = $donation_box[$locale] ?? $donation_box['en'] ?? (is_string(reset($donation_box)) ? reset($donation_box) : ''); } @endphp

With this block, the system now: first tries to use Spatie's native method, and as a fallback extracts the JSON value manually using the current site language.

🏆 Fix #3 — Certification Logos FIX

The certification icons section (Candid, Best NGO, etc.) tried to access $cert->image and build paths from the storage/sponsor/ folder, but the Sponsor model saves the logo in the company_logo field and the correct folder is storage/company_logo/.

Before: asset('storage/sponsor/' . $cert->image)

After: asset('storage/company_logo/' . $cert->company_logo)
The alt attribute was also added using $cert->company_name for accessibility.

📝 Fix #4 — Duplicate Description FIX

The condition for the short_description field had an unnecessary double fallback: $cam->short_description ?? $cam->description. This caused the intro text to appear twice in a row on the screen because the description content was also printed further down in another section. It was simplified to check only short_description.

📊 Improvement #5 — OG and Twitter Meta Tags IMPROVEMENT

The Open Graph (Facebook/WhatsApp) and Twitter/X meta tags also used $cam->cover_image_url and $cam->og_image_url, which only exist during the admin preview. They were updated to use the $cover_image_url and $og_image_url variables correctly built at the top of the file, ensuring images are displayed when sharing on social networks.

7. Additional Technical Notes
  • Model used: App\Models\MiniCampaignTemplate — uses the Spatie\Translatable\HasTranslations trait. The translatable fields are saved as JSON in a single database column.
  • Public controller: HomeController@miniCampaign — passes the model instance as $cam to the view. It does not pre-process the fields, so the view now does that work.
  • Preview controller (admin): MiniCampaignController@showPreview — passes a stdClass with the fields already processed (cover_image_url, og_image_url). The new template code is compatible with both sources thanks to the isset() block.
  • Storage folders: Images are located in storage/app/public/cover_image/ and are accessible via the symlink in public/storage/.
